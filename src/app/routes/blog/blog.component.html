Why Single Page Applications are the way forward

New technologies for creating clean single page applications have been emerging over the last few years. We've seen Google release the AngularJS framework which was followed by the newer Angular versions while Facebook has developed the React framework. Both of these frameworks are now at the forefront of web development.

What is a Single Page Application?

SPAs are web applications that are loaded within the browser where minimal reloading is required. You've probably used a SPA without even realising. Google's Gmail applicaion is a perfect example of a single page application. Once the initial page has loaded, it is initialized, the screen is loaded, and then all subsequent changes happen seamlessly without reloading the entire page. The bonus of using SPAs means that assets and resources such as images are only loaded once resulting in websites that will load quicker and perform better even on the slowest of connections.

Pros
  • Fast - HTML, CSS & Scripts are loaded once
  • Reusable - You can build smaller components to make up larger pages while reusing the components elsewhere
  • Data is loaded on demand - only the data your require to display is loaded through asyncronous calls.
  • We can use more animations to provide a much better user experience.
  • Mobile Friendly - We can use the same backend code with a mobile design rather than re-writing for the specific purpose
Cons
  • SEO is tricky - Routes are dynamically rendered it's harder for web crawlers to scan your site. A solution to this is to use server side rendering (a topic for another day)
  • Javascript is required - Although most browsers come with javascript activated, if it isn't, your page just wont work
  • Browser buttons may not work well - due to the nature of SPAs the back and forward functionality of your browser may not work as expected.
Final Word

I believe SPAs provided the end user with a much more rich user experience compared to that of a multi-page applications. Page / Data loading times could inhibit the user.
